A SCADA upgrade in Australia should start with an audit of current architecture, licensing, and data flows. This helps define a low-risk migration sequence.

## Audit current state

Document servers, clients, PLC connections, historians, and report dependencies.

## Design target architecture

Define redundancy, alarm strategy, cybersecurity zones, and integration endpoints.

## Test before cutover

Use staging environments and formal FAT to reduce risk during production windows.
